Zone Distribution Methodology

Zone-weighted costs are calculated from actual order data: each customer's orders are mapped to freight zones and a per-customer zone distribution is computed.

Weighted Cost
Cost = Σ(zone% × zone cost) from actual order distribution
Worst Cost
Uses last zone (farthest / most expensive zone)
RegionZonesServices
UK4 zones: Mainland, Highlands, N. Ireland, Channel IslandsEconomy, PriorityCity name → zone
EUVariable per service typeInt'l Economy, Int'l Express, Domestic, Int'l Economy ReturnCountry code → zone
CA16 zones (D01-D16)Express, GroundFSA (first 3 chars) → zone

Zone mapping: UK uses delivery city → zone (828 cities mapped from order data; N.Ireland, Highlands, and Channel Islands cities identified via postcode cross-reference + manual overrides). EU uses country → zone from rate card (~30 countries). CA uses Forward Sortation Area → zone (903 FSAs). UK/CA returns use outbound zone distribution as proxy. Default: Zone 1 (Mainland) for unmatched cities.

Return zones: UK/CA returns use the customer's outbound zone distribution as proxy. EU returns use country → return zone mapping.

Freight lookup: Finds the largest weight tier ≤ the shipment weight. If an add-on/kg rate exists for that tier, excess weight is charged pro-rata on top of the base rate.

How shipment weights are derived from customer data before freight lookup.

RuleDetail
Weight formularound(avg_weight_per_item × avg_items_per_order)
Minimum weight1 kg (UK / EU) or 1 lb (CA)
CA conversion1 kg = 2.20462 lbs — CA freight tables use pounds
UK / EU unitKilograms (kg)
RoundingStandard rounding (Math.round)

Margin Calculation Formulas

How selling prices and margin percentages are computed across the Pricing Model and Margin Analysis sections.

Selling Price
Cost ÷ (1 − Margin%)
e.g. £19.73 at 30% = £28.19
Weighted Margin
(Fee − Wtd Cost) ÷ Fee × 100
Uses zone-weighted freight
Worst Margin
(Fee − Worst Cost) ÷ Fee × 100
Uses last zone freight rate
